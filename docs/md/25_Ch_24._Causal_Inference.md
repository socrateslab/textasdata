# Chapter 24. Causal Inference

Causal Inference
在过去二十年中，计算机科学家和统计学家在开发和生产用于分析自然语言的方法方面取得了巨大进展。这些模型中有许多的目标都是预测一个结果变量，我们在上一章中已经介绍过这一点。例如，已经开发出预测模型来识别垃圾邮件、预测网站用户可能点击的广告，或通过搜索向用户推送相关链接。这些模型使用带标签的数据集来学习预测变量与结果变量之间的关系，以便做出决策。
尽管预测对于在网页界面上向个体提供特定内容很有用，但社会科学家常常对能为一国社会政策提供信息的问题感兴趣——即做出能为政策制定者提供信息的决策。例如，社会科学家可能有兴趣了解竞选策略的改变——例如竞选广告中的"负面攻击"（going negative）——会如何影响对候选人的整体支持度。或者，他们可能有兴趣了解社交媒体上的错误信息如何影响公众对选举公平性的认知。这些问题关注的并非准确预测未来——即将会发生什么——而是关注估计在世界另一种情形（反事实）下本会发生什么。
由于分析自然语言的方法通常并非为因果推断而设计，当社会科学家开始把这些方法应用于研究社会现象时，他们首先将这些技术用于发现和测量的描述性目的，这些任务我们在之前的章节中已有描述。例如，在社会学和心理学中，这些方法被用来追踪美国政府讨论艺术与文化的方式 (DiMaggio, Nag, and Blei, 2013)[^1]，以及语言如何反映人格与社会互动 (Schwartz et al., 2013)[^2]。在政治学中，文本方法被用来测量联邦储备委员会中的审议过程 (Bailey and Schonhardt-Bailey, 2008)[^3]，研究文本作者的意识形态 (Slapin and Proksch, 2008; Benoit and Laver, 2003; Lowe, 2008)[^4][^5][^6]，以及评估社交媒体帖子的情感 (Hopkins and King, 2010)[^7]。
社会科学中一些最早使用文本作为数据进行发现和测量的工作，明确地将元数据纳入测量模型，以探索不同数据子集之间测量的变异。例如，Grimmer (2010) 的"表达议程模型"（Expressed Agenda Model）将一组帖子的作者信息明确地建模进一个主题模型，估计某一议题被讨论的程度如何随每个作者个体而变化。Grimmer (2013b) 扩展了该模型，以探究美国国会议员在与选民交谈时，如何根据其特征（例如其选区的竞争性）而关注不同的议题。Quinn 等人 (2010) 创建了一个动态主题模型，能够测量主题如何随时间变化，并用它来研究美国参议院七年间讨论的变化。这些模型与计算机科学中大量探索在测量模型中将元数据与文本结合使用的文献共同发展，例如 Mimno and McCallum (2008) 以及 Eisenstein, Ahmed, and Xing (2011)[^8]。
将元数据纳入测量模型的能力，启发了进一步的研究，探索文本作为数据的方法如何能与因果推断结合使用。如果元数据可以被纳入文本的测量模型，那么实验中处理（干预）组和对照组状态的指示变量是否也可以被纳入，以估计处理对文本结果变量的影响？Roberts 等人 (2014)[^9] 以及 Roberts, Stewart, and Airoldi (2016)[^10] 探讨了一个我们在第13章回顾过的通用模型——结构主题模型（Structural Topic Model），用于将元数据纳入文本，并展示该模型如何用于估计以文本为结果变量的实验中的处理效应。Gill and Hall (2015) 展示了如何使用置换检验来检验随机化处理对文本结果变量的影响。这些论文朝着定义问题并应对将实验与非结构化文本结合使用的挑战迈出了第一步。
在本章中，我们向前迈出一步，明确界定在因果推断设计中，当文本作为处理（干预）、结果变量或混杂因素的测量时所必需的研究过程和假设。我们建立在先前定义用文本进行因果推断问题的工作之上，明确概述在因果分析中使用文本的机会，以及用文本这类高维数据进行因果推断所面临的挑战。
我们在本章开头先简要概述因果推断，并再次回顾预测、测量与因果推断之间的区别。然后我们引入使用文本数据进行因果推断的五条指导原则，并在本章余下部分详细回到这些原则。接着我们概述因果推断的基本问题以及文本数据给因果推断带来的挑战，并提出应对这些问题的方案。第25章和第26章提供了在实验中将文本用作结果变量和用作处理（干预）的几个示例。我们在第27章还会探讨在观察性研究中以文本为条件。本章不仅大量借鉴了社会科学中将文本数据与实验结合使用的前期工作，也借鉴了配套论文，即 Egami 等人 (2018)、Fong and Grimmer (2021)[^11]，以及 Roberts, Stewart, and Nielsen (2020)[^12]。

### 24.1 Introduction to Causal Inference

24.1  Introduction to Causal Inference
我们在本章开头对因果推断做一个基础介绍。因果推断的目标是识别处理（干预）变量 t_i 对结果变量 y_i 的效应。这一框架足够通用，可以应用于社会科学中许多不同的因果过程——从竞选期间投放的负面广告对候选人得票率的影响，到党派性新闻消费对研究参与者政治观点的影响。t 可以取几种不同状态中的一种；换句话说，存在不同的处理，我们希望估计接受不同处理的单元之间结果 y 的差异。关于我们所借鉴的因果推断的优秀导论，参见 Morgan and Winship (2015)。
我们从最简单的例子开始，假设 t_i 只取两种状态——处理（干预）和对照。我们观察——通过实验的或观察性的方式——两组个体，一组接受了处理，一组接受了对照。驱动因果推断理解框架的概念是：无论一个单元接受的是处理还是对照，每个观测都有两个潜在结果，即结果变量 y_i 的潜在值：如果它接受了处理会发生什么，以及如果它接受了对照会发生什么。通常，我们将这些潜在结果记为 y_i(1) 和 y_i(0)，即个体 i 接受处理（t_i = 1）时的结果，或个体 i 接受对照（t_i = 0）时的结果。
因果推断的目标是描述潜在结果之间的差异——单元 i 在处理下的结果将如何不同于在对照下的结果。对于任何个体，处理 t_i 的因果效应是不同 t 状态下的潜在结果之差，即 y_i(1) − y_i(0)。然而，每个个体只能接受一种处理。因此，我们只能观察到 y_i(1) 和 y_i(0) 中的一个。我们观察到的那个结果通常被称为事实（factual），而我们没有观察到的那个结果通常被称为反事实（counterfactual）。因果推断的根本问题（fundamental problem of causal inference）在于我们只能观察到事实而无法观察到反事实 (Holland, 1986)[^13]。因此，我们无法直接观察到个体因果效应。
即便如此，我们确实观察到了两组个体，一组接受了处理，一组接受了对照。在某些情况下，我们可以利用这两组来理解因果效应。首先，我们将关注的数量改为平均因果效应 ȳ_T − ȳ_C，其中 ȳ_T = 1/N ∑_{i=1}^N y_i(1)，ȳ_C = 1/N ∑_{i=1}^N y_i(0)。我们无法直接估计 ȳ_T 或 ȳ_C 中的任何一个，因为我们没有对所有 i 都观察到 y_i(1) 或 y_i(0)。
然而，我们可以将平均因果效应 ȳ_T − ȳ_C 分解为反映我们已有数据的组成部分。设 p_T 为样本中处理组所占比例，T 收集所有被处理的单元，C 收集所有对照组单元；则有：
ȳ T − ȳ C = p T ( ȳ i ∈ T T − ȳ i ∈ T C ) ︸ Avg. Effect on Treated + ( 1 − p T ) ( ȳ i ∈ C T − ȳ i ∈ C C ) ︸ Avg. Effect on Control = p T ȳ i ∈ T T − ( 1 − p T ) ȳ i ∈ C C ︸ observed terms + ( 1 − p T ) ȳ i ∈ C T − p T ȳ i ∈ T C ︸ unobserved
我们确实观察到 ȳ_i∈T_T，即处理组上处理（干预）的平均结果，以及 ȳ_i∈C_C，即对照组上对照的平均结果。我们没有观察到 ȳ_i∈T_C 或 ȳ_i∈C_T，即这两组中每一组的反事实。
这一分解揭示了我们可以用已有的数据来估计我们所没有的数据。如果我们想用可观察数据 ȳ_i∈T_T − ȳ_i∈C_C 来估计 ȳ_T − ȳ_C，这只有在 ȳ_i∈T_T = ȳ_i∈C_T 且 ȳ_i∈T_C = ȳ_i∈C_C 时才有效。换句话说，如果我们能用处理组在处理状态下的观测结果来替代对照组在处理状态下缺失的结果，反之对对照状态亦然，那么我们就能够在没有缺失潜在结果的情况下估计平均因果效应。
什么时候会有 ȳ_i∈T_T = ȳ_i∈C_T 且 ȳ_i∈T_C = ȳ_i∈C_C？这只有在潜在结果不依赖于个体是否被分配到处理（干预）时才会发生，即 ȳ_i∈C_T 不会因为适用于对照组而与 ȳ_i∈T_T 有某种系统性差异时。可能出什么问题？首先，我们目前所呈现的符号本身依赖于每个单元有两个潜在结果（每种处理状态下一个）这一观念。稳定单元处理值假设（SUTVA）将这一假设形式化，它表明：当我们观察到某一给定处理状态下的结果时，它反映了该单元在该处理状态下的反事实。例如，SUTVA 排除了单元之间的干扰——一个单元的处理不会影响另一个单元的响应。它还假设处理在各单元间是一致的——一组不会接受与另一组不同类型的处理。如果 SUTVA 不成立，那么一组的结果可能反映出另一组的处理，这意味着我们无法以通常的方式估计平均因果效应。举例来说，假设我们有兴趣了解看到一则新闻故事对支持某位政治候选人产生的影响。我们在社交媒体上与处理组分享这则新闻故事，但不与对照组分享。但随后处理组分享了这则新闻故事，使得对照组的成员也看到了它。对照组不再是处理组的一个好的反事实，因为它不再反映处理组如果没有看到该故事会发生什么。
其次，分配机制可能是混杂的（confounded）。也就是说，处理与结果之间的关联可能不仅反映因果效应，还反映某种非因果的关联。如果对照组与处理组在某种与潜在结果相关的方式上有所不同，那么对照组在对照分配下的结果就不能用作处理组在对照分配下结果的估计。为了能够用观测数据估计平均因果效应，我们需要可忽略性（ignorability），即潜在结果需要独立于处理分配：(y_i(1), y_i(0)) ╨ t_i。举例来说，假设研究者允许人们选择加入以接收新闻故事，因此对政治感兴趣的人自我选择了进入处理组。处理组和对照组不再具有可比性（处理组从一开始就更有政治性），并且处理（决定接收新闻故事）与潜在结果相关。
科学家应对混杂的一种特别有力的方式是处理分配的随机化。如果处理组和对照组是基于随机数生成器来分配的，那么平均而言，跨实验处理组将与对照组相似，并且一组的结果可以替代另一组的潜在结果。在随机化下，处理组和对照组不太可能以与结果相关的系统性方式出现差异。随机化确保了处理分配独立于潜在结果，因为单元无法自我选择进入处理。
如果满足 SUTVA 和可忽略性的条件，简单线性回归中的参数就对应于平均处理效应。1 将 y 对 t 回归得到方程：
y_i = β_0 + β_1 t_i + ε_i。
由该回归，平均处理效应的各组成部分为：
ȳ_i∈T_T = E[y_i | t_i = 1] = β_0 + β_1 + E[ε_i | t_i = 1]
ȳ_i∈C_C = E[y_i | t_i = 0] = β_0 + E[ε_i | t_i = 0]。
只有当误差与处理状态不相关时，ȳ_i∈T_T − ȳ_i∈C_C 才会等于处理效应 β_1，这等同于说处理分配与潜在结果不相关。使用实验进行因果推断的研究者通常会用回归来估计平均因果效应。

### 24.2 Similarities and Differences between Prediction and Measurement, and Causal Inference

24.2  Similarities and Differences between Prediction and Measurement, and Causal Inference
尽管我们到目前为止为描述因果推断的目的做了大量概念性工作，我们最终落脚于一个简单的线性回归，这与我们在测量章节和预测章节中描述的方法颇为相似。事实上，在测量和预测章节中描述的有监督学习问题中，我们也从一个简单线性回归开始，其中 y_i 是所关注的类别或预测，而 W_i 是我们用来预测该类别或结果的文档中的词元。
在因果推断中，我们感兴趣的是如果有一个或一组自变量发生变化，会发生什么或会有什么不同。本会发生什么——即反事实——与实际上发生了什么——即结果——之间的差异，是我们所关注的因果估计量（causal estimand），我们希望无偏地估计它。与预测不同，我们常常不太在意我们的模型是否准确预测了结果，因为我们观察到了结果；相反，我们想预测潜在结果，即如果有一个或一组自变量不同，本会发生什么的最佳猜测。
因果推断任务与预测任务之间的差异，对我们在每种情况下如何处理数据有着重要影响。事实上，我们认为关于因果推断和预测的许多混淆源于将二者混为一谈。在预测中，正如我们在上一章中所讨论的，只要模型的预测结果准确，我们就不太关心模型内部包含了什么。在预测中，我们乐意给模型中的系数引入偏差，只要它能提升我们的预测能力。因果推断中则不然，我们必须非常小心进入模型的每个变量在因果过程中的作用，以确保我们估计的是本会发生什么，重要的是避免处理后偏差（post-treatment bias）并考虑潜在的混杂。
类似地，尽管我们建议在因果推断和预测中都使用分样本方法进行验证，但分样本在两种情况下用途不同。在预测中，我们使用分样本来估计错误率并调优模型。也就是说，我们重新审视模型并重新调优，直到将测试数据中的泛化误差最小化。因果推断中则不然，在因果推断中，分样本用于识别，并确保我们正确地刻画处理效应或所关注系数周围的不确定性。因为我们认为预测与因果推断之间的区别非常重要，我们将在本章中反复回到这一主题。

24.3  Key Principles of Causal Inference with Text
24.3.1  THE CORE PROBLEMS OF CAUSAL INFERENCE REMAIN, EVEN WHEN WORKING WITH TEXT
使用文本数据并不能使我们从因果推断的核心挑战中解脱出来。即使是文本这样的新数据，我们也会遇到因果推断的根本问题：我们无法观察到个体观测在不同处理条件下本会发生什么 (Holland, 1986)[^13]。为了准确估计反事实——即如果世界状态不同，某个结果的值——我们需要任何良好因果推断所需的相同基本要素。我们需要一个好的设计，关注我们所使用的样本，并在依赖"基于可观测变量的选择"（selection on observables）假设时保持谨慎。混杂、反向因果以及观测之间的依赖，在我们有文本数据的情况下与没有文本数据的情况下一样有可能发生，我们需要对这些以及其他对推断的威胁保持警惕。
24.3.2  OUR CONCEPTUALIZATION OF THE TREATMENT AND OUTCOME REMAINS A CRITICAL COMPONENT OF CAUSAL INFERENCE WITH TEXT
在发现章节中，我们说明了分析者如何通过探索文本的不同表征来从文本数据中发现新概念。在测量章节中，我们描述了研究者如何使用有监督机器学习在大规模上对文本中的给定概念进行测量。在本章中，我们展示如何利用这些概念进行因果推断。然而，与前面各章不同，我们现在必须特别小心我们是如何得出概念化的——特别是我们在发展概念化时是否使用了关于处理和对照的信息。我们将强调在因果推断框架中发展和发现概念时的两个主要风险。第一个风险应当让人感到熟悉——我们担心会对文本过拟合（overfit）。也就是说，一个风险是我们将发展出一种捕捉文本非系统性层面的分类方案，本质上是在文档中发现短暂易逝的噪声。当测量或发现中出现这种情况时，风险在于我们将注意力分配给那些极其罕见且不太可能出现在其他样本中的类别。因果推断中的过拟合更为有害，因为它会导致我们在干预其实没有系统性效应时，推断出干预具有系统性效应。
第二个风险是我们称之为"潜在变量因果推断的根本问题"（Fundamental Problem of Causal Inference with Latent Variables, FPCILV）的问题 (Egami et al., 2018)[^14]。FPCILV 发生在分析者通过参考实验中处理的结果来发展映射函数，从而在单元之间制造依赖时。如果映射函数是用与最终分析相同的数据发展起来的，那么一个单元的估计潜在结果将依赖于其他单元的处理状态，因为映射函数依赖于其他单元。这种依赖违反了 SUTVA——因果推断的必要条件之一。我们将在本章后面更详细地解释过拟合和 FPCILV。
#### 24.3.3 The challenges of making causal inferences with text underscore the need for sequential science

24.3.3  THE CHALLENGES OF MAKING CAUSAL INFERENCES WITH TEXT UNDERSCORE THE NEED FOR SEQUENTIAL SCIENCE
社会科学是创造可推广知识以解释或预测社会模式的过程。社会科学中最好的因果推断，或解释，源于理论与严谨的经验证据之间的紧密关系。这种互动是序贯的——理论随着我们从实验中学习而更新，而我们运行的实验由理论指导。因为这种互动是序贯的，证据的获取永远不会完全完成，也没有哪个实验是完全决定性的。在不同时期或不同人群中对实验的复制，扩展了我们的证据基础，从而使我们能够得出更准确的理论。这些理论反过来又提示了下一个实验。
由于实验与理论之间的相互作用，许多人建议，在进行实验之前，应列出实验中的所有参数，包括实验将要研究的变量、这些变量如何被测量，以及实验应当报告的最终检验 (Humphreys, Sanchez de la Sierra, and Van der Windt, 2013)[^15]。用本书的话说，映射函数必须在实验进行之前被明确指定。这一建议背后的直觉是：如果一个实验真正检验的是事先已知的理论，那么实验在启动后不应改变，并且在预设研究设计之外报告结果可能会损害分析。许多人建议研究者在开展实验之前预先登记（pre-register）他们的实验，制定一个预先分析计划（PAP），以束缚研究者的手脚，使他们在实验运行之前无法更改数据收集和分析。
尽管这种数据分析中的僵化在特定情境下可能非常有用，正如我们将在下面描述的，但它依赖大量确定性，关于我们可能使用的变量、这些变量如何被测量，以及我们正在考察的数据的属性。它依赖于实验平稳运行，没有预料之外的波折。当然，在运行实验之前手中掌握所有这些信息，或者实现过程中的意外事件不会改变研究设计，这种情况很少见。在处理像文本这样的高维数据时尤其如此，研究者可能无法预见受试者将表达的主题和情感。预期实验将如何运行的困难，可能导致 PAP 的尴尬应用，并导致损害其有效性的制度的产生。例如，现在常见的情况是，PAP 被修订以纳入新数据，或在与合作组织的经验之后被更改。而且因为不可能预见 PAP 的所有问题，研究者正试图在面对偏差时制定标准操作程序。
一旦我们能够稍微改动，我们就为许多 PAP 在最理想情况下本能够防范的同样弊端打开了大门。
我们主张不要依赖完美的先见之明来预测研究设计中的潜在偏差，而是在每个实验内部开展理论与实证的相互作用。也就是说，我们的研究设计鼓励用户将文本和其他数据随机划分为训练集和测试集。尽管这种划分在机器学习中很常见且本书通篇使用，我们在这里使用它是出于不同的原因。首先，如果我们手中已有一个映射函数，我们可以用训练集来排除与我们预期的数据收集工作之间的任何偏差。我们可以验证我们想要测量的确实被测量到了。完成这一点之后，我们就可以写下一份定义良好的 PAP 并分析测试数据。其次，当我们事先不知道映射函数时，我们可以用训练数据去发现它，探索我们的数据以确保我们揭示出一个对数据集有用的编码方案。有了这个编码方案在手，我们就能定义一套清晰、易于应用于测试数据且不会有意料之外波折的规则。正如我们将在本章中展示的，使用样本划分（sample splitting）确保我们避免 FPCILV 问题，限制我们过拟合的机会，并帮助我们可信地约束研究者的自由度（researcher degrees of freedom）。我们认为，这比在收集任何数据之前就执行预先分析计划要更少浪费，因为它允许研究者为数据定义最有用的测量、发现未预料到的模式，并考虑到实验的实际实施，同时仍然确保从数据中得出的最终推断是可信的。
这最后两条原则的讨论核心是映射函数——我们如何发现它，以及我们在何时将其固定以进行因果推断。因此，我们在本章开头先回顾这个函数——为什么需要它，我们如何发现它，以及我们希望它具有什么性质。

### 24.4 The Mapping Function

24.4  The Mapping Function
第3部分（发现）和第4部分（测量）的核心任务是创建从文本到文本的某种低维摘要的映射。这个映射函数，Egami 等人 (2018) 称之为 g，是使用文本作为数据的基础，因为它将文本复杂且高维的性质简化为对我们研究问题至关重要的文本本质的低维提炼。这个映射使我们能够进行后续分析，将我们的测量与其他变量联系起来。
在本书中，我们已经看到映射函数有多种形式。Catalinac (2016a) 使用潜在狄利克雷分配（Latent Dirichlet Allocation）主题模型，创建从日本选举宣言到候选人对外交政策关注度测量之间的映射。Gillion (2016) 用一个有监督分类器测量政治中与种族相关的论述，该分类器以总统和国会演讲作为输入。Nielsen (2017) 使用穆斯林神职人员写的文本，基于朴素贝叶斯分类器创建一个量表，构建映射函数，以此生成神职人员对圣战支持度的测量。这些映射函数都相当多样——我们已经看到使用从手工编码、字典方法、有监督方法、无监督方法到标度方法等工具发展起来的映射函数。这些函数创建的测量有时是连续的，有时是类别的。但在每种情况下，函数都将文本简化为一个所关注的测量。
我们强调，尽管从文本创建测量本质上需要将文本映射为文本的简化，但各种形式的映射函数几乎出现在每一项研究之中，包括那些不使用文本的研究。例如，当我们使用一个指数来衡量一个国家民主的程度，比如政体指数（Polity Index），我们是将关于一个国家制度的大量信息压缩到一个数字——"Polity"上。尽管在量化分析中使用民主概念时如果不对其维度做某种程度的约简将很难操作，但我们应该始终力求理解我们的测量在多大程度上真正反映了我们有兴趣研究的概念。
在我们这种不可知论（agnostic）的文本分析方法中，我们一直强调，对于给定的文本集合，不存在唯一正确的组织方式；相反，我们对文本所做的提炼必须针对研究者的目标。我们产生的映射函数并不是个体如何在我们的语料（库）中生成文本的一个模型，而更多是一个生成所关注测量的工具，它由研究问题所驱动。因为这种方法论的不可知性，在第3部分（发现）中，我们鼓励研究者探索他们的数据，通过应用不同的映射函数来考察文本的新组织方式，从而引导他们产生新的想法，进而产生新的概念和测量。在第4部分（测量）中，我们鼓励研究者在迭代过程中精炼他们的测量，迭代地开发用于手工编码的编码手册和用于有监督学习的分类器，以最佳地捕捉他们想要测量的概念。在这两个部分中，我们都强调了最终测量广泛验证的重要性。
尽管我们在全书倡导探索并学习从文本到所关注概念的映射，但发展一个映射函数的过程可能给因果推断带来复杂性。在本节中，我们遵循 Egami 等人 (2018)，先将映射函数引入上面的因果推断符号中。与 Egami 等人 (2018) 一样，我们称映射函数为 g。然后我们描述在同一分析中既发现 g 又用 g 估计因果效应所存在的问题。最后，我们引入一个包含样本划分流程的工作流，以应对这一问题。
#### 24.4.1 Causal Inference with g

24.4.1  CAUSAL INFERENCE WITH g
正如我们在第24.1节中所讨论的，为了识别平均处理效应，我们做出几个重要的假设。首先，我们做出稳定单元处理值假设（SUTVA），它要求每个单元的潜在结果不受另一个单元处理状态的影响。其次，我们假设可忽略性，即每个单元的潜在结果独立于处理分配。最后，我们假设正向性（positivity）——所有单元至少都有一些接受处理的概率。我们将处理的可能取值集合称为 ᷘ。
Assumption 1 (SUTVA) For all individuals i, y i ( t ) = y i ( t i ) .
Assumption 2 (Ignorability) y i ( t ) ╨ t i for t ∈ ᷘ.
Assumption 3 (Positivity) P r ( t i = t ) > 0 for t ∈ ᷘ .
为了解释发现 g 给因果推断带来的挑战，我们将 g 引入因果推断符号。为了同时考虑作为文本的结果变量和处理，我们考虑两种不同的应用：1) 处理是二元的且文本测量结果变量，我们称之为"文本作为结果变量"（text as outcome）；2) 处理是文本而结果变量是一个标量，我们称之为"文本作为处理"（text as treatment）。在第一种情况下，结果变量 y_i 是多维的，而在第二种情况下，处理 t_i 是多维的。我们在第25章和第26章中更详细地介绍每种情况的特定示例。
暂时假设在每种情况下——文本作为处理和文本作为结果变量——映射函数 g 是已知的。也就是说，我们已经决定了将用于分析的文本压缩方式。我们称这种文本的压缩为 π_i（i = 1, …, N），即单元 i 的 π 值。这种压缩在 ᷔ 空间中取值。该压缩可以是我们在本书中迄今讨论过的任何文本表征——它可以表示手工编码文档的类别、来自主题模型的文档主题比例，或由字典或有监督学习算法生成的量表。在文本作为结果变量的情况下，g 将 y 映射到 ᷔ，即 g: ᷝ → ᷔ_y，且 g(y_i) = π_i。在文本作为处理的情况下，g 将 t 映射到 R，即 g: ᷘ → ᷔ_t，且 g(t_i) = π_i。为了概念上的简便，在文本作为处理的情况下我们通常将 ᷔ 视为一组 K 个二元的特征向量 (Fong and Grimmer, 2016; Egami et al., 2018; Fong and Grimmer, 2021)[^16][^14][^11]。
使用 g，我们可以重写我们在第24.1节定义的平均处理效应，但这次把 g 包括进来。首先考虑二元处理、结果由 g 映射到 K 个类别之一的文本作为结果变量的情况。然后我们可以将潜在结果重新定义为：在处理下 g(y_i(1)) = π_i(1)，在对照下 g(y_i(0)) = π_i(0)，其中 π_i(1) 是结果在处理下会取的 K 个类别的向量。我们将 π_i^k(1) 定义为第 k 个类别在处理下的潜在结果。那么平均处理效应可以重写为：
ATE_k = E[g(y_i(1))_k − g(y_i(0))_k]   (24.4.1) = E[π_i^k(1) − π_i^k(0)]。
#### 24.4.2 Identification and Overfitting

24.4.2  IDENTIFICATION AND OVERFITTING
在上一节中，我们假设 g 已经由研究者创建好。然而，在本书大部分内容中，我们不仅描述了发现和创建 g 的过程，还鼓励对 g 进行探索和迭代。然而，在因果推断的情境下，在同一分析中既发现 g 又估计它可能会有问题。Egami 等人 (2018) 描述了在估计因果效应时发现 g 的两个主要问题：识别（identification）——Egami 等人 (2018) 称之为"潜在变量因果推断的根本问题"（FPCILV）——以及过拟合。
Fundamental Problem of Causal Inference with Latent Variables
为了使 SUTVA（假设1）成立，每个单元的潜在结果不应依赖于其他单元的处理状态。然而，当我们使用数据来发现 g 时，映射函数 g 依赖于所有单元的文本。例如，在主题模型中，我们利用文档之间的相似性来寻找倾向于在文本中共同出现的词群。在有监督学习方法中，算法使用所有文本来学习带有不同标签的文档之间的差异。即使在手工编码中，我们也基于在文档中看到的差异来发展编码方案。
这意味着，如果文本不同，我们很可能就会发现一个不同的映射函数。推而广之，每个文档的 π_i 也就依赖于所有其他单元的文本。在文本作为结果变量的情况下，这意味着每个单元的潜在结果 π_i(1) = g(y_i(1)) 和 π_i(0) = g(y_i(0)) 会受到其他单元处理分配的影响，因为它们依赖于 g。在文本作为处理的情况下，这意味着处理本身通过映射函数相互影响。在这两种情况下，这都给因果推断造成了一个根本性问题，因为潜在结果如果不参照其他单元就无法定义。关于该问题更技术性的说明可见于 Egami 等人 (2018)。
Overfitting
在同一分析中既发现 g 又估计因果效应，也可能导致过拟合，即研究者发现可能不能推广的关系。过拟合在任何研究设计中都可能是个问题，但因为文本维度极高，在文本上问题更为尖锐，因为可能会发现许多可能的 g。以文本作为结果变量的情况为例，研究者想测量处理对文本情感的影响。设想纯粹出于偶然，处理组文本中不成比例地大量使用了 sunny（晴朗）这个词。研究者可能没有将其纳入一个先验的正面情感字典中，但在寻找 g 的过程中可能因为它与处理相关而兴奋，从而将其选作一个测量——这是一种钓鱼（fishing）行为 (Simmons, Nelson, and Simonsohn, 2011)[^17]。或者，研究者可能因为这个词的含义而将其纳入字典，即使她并未意识到它与结果的关系。无论哪种情况，这一发现都无法在该数据集之外推广，因为 sunny 与处理的相关是偶然的而非系统性的。当存在许多特征（如文本数据那样）时，其中至少有一些会偶然地与处理相关，这使得研究者防范过拟合尤为重要。

24.5  Workflows for Making Causal Inferences with Text
在本节中，我们介绍用文本进行因果推断的不同方法，以应对上面提出的两个关注点——FPCILV 和过拟合。在下面每种方法中，重要的特征都是将发现 g 与估计因果效应分开。
24.5.1  Define g before Looking at the Documents
将发现 g 与估计因果效应分离的最简单方法，是事先定义 g。例如，正如我们在第16章中所讨论的，研究者可以使用一个已有的字典作为 g。或者，正如我们在第18章中所讨论的，研究者可以使用一个已有的编码手册，而不是自己定义。在这些情况下，我们都可以直接在文本作为结果变量或文本作为处理的示例中估计因果效应。然而，我们在本书中也指出了这种方法的缺点。在第16章中，我们描述了在一个语料（库）上开发的现成字典可能无法很好地推广到另一个语料（库），如 Loughran and McDonald (2011) 所示。这一点也适用于现成的编码手册。以 Boydstun (2013) 为例，她使用了既有的"政策议程编码手册"（Policy Agendas Codebook）来为报纸文章编码，正如我们在第18章中所描述的。即使使用既有编码手册，她也对编码手册进行了更新和注释，以便能够准确地应用于她的文档。如果不更新，她对报纸文章的标签最终会不够准确。
事实上，使用现有的现成测量，牺牲了我们视为文本作为数据之前景（promise）的许多东西。我们在本书中描述的所有发现和测量都来自对概念的探索与精炼。如果在每次分析中都事先指定 g，我们将错过重要的发现和探索成果。即使我们在事前对想要测量的概念有所设想，在查看数据之前用 g 将该测量操作化，也可能引导我们得到一个对真正所关注概念不令人满意的代理变量。
24.5.2  USE A TRAIN/TEST SPLIT
另一种方法是将样本划分为两组——一组用于发现和精炼 g，另一组用于估计因果效应。这种方法由 Fong and Grimmer (2016) 和 Egami 等人 (2018)[^14] 提出。样本划分解决了 FPCILV 问题，因为潜在结果是在测试集中定义的——在测试集中，g 不依赖于任何单元的处理状态。它同时也通过确保训练集中发现的任何关系都能推广到测试集，来解决过拟合问题。这样做的好处是，我们可以用训练集来探索和精炼 g，并且知道我们将能通过测试集估计最终 ATE 来防范过拟合。
工作流程如下：
1. 划分样本。 在运行实验之后，研究者首先将样本划分为两部分，一部分用于发现（训练集），一部分用于估计因果效应（测试集）。将多少数据放入训练集而非测试集，取决于发现并精炼 g 所需的数据量与估计精确效应所需的数据量之间的权衡。选择哪种划分必然因应用而异，但无论使用哪种划分，我们都建议随机地选取训练文档。重要的是，测试集应被放在一个在 g 估计出来之前不被触碰或查看的地方。测试集只能使用一次——用于估计因果效应。因此，这种训练/测试划分版本与我们使用交叉验证来训练分类器（我们在第20章中讨论过）相当不同。在这种情况下，交叉验证会生成多个不同的测量（每个折一个），并最终在这组测量上违反 FPCILV。相反，我们这里建议的程序更类似于我们在第21章描述的训练/测试划分，其中测试集只使用一次——用于最终测量。
2. 发现 g。 在划分样本并移除测试集之后，研究者可以开始发现和精炼 g 的过程。在这里，我们可以使用本书中描述的所有方法来探索、发现、测量并验证我们想要使用的概念。我们希望找到一个对研究问题有用且可推广的测量——一个我们可能在其余训练数据之外其他数据集中也能发现并能够测量的测量。
3. 验证 g。 在确定一个 g 之后，我们必须验证它，正如我们在第3部分（发现）和第4部分（测量）中所描述的，特别是第12、20和21章。验证确保我们确实测量了我们所声称要测量的概念，并且我们的准确度高。验证通常也要求将训练集进一步划分为若干部分，这出现在交叉验证（第20章）或验证用于测量的无监督方法（第21章）的过程中。
4. 应用于测试集并重新验证。 因为我们只能使用测试集一次，所以在打开测试集之前，我们必须确保我们对 g 满意，并且 g 的所有方面都已良好定义，包括我们将如何在测试集中表征文本，以及我们将如何应用映射函数以获得标签。在这一点上，我们甚至可以撰写一份预先分析计划或声明我们的设计，以确保我们有一份计算 ATE 的计划 (Gerber and Green, 2012; Humphreys, Sanchez de la Sierra, and Van der Windt, 2013; Blair et al., 2019)[^18][^15][^19]。然后我们将 g 应用于测试集并估计平均因果效应。但我们尚未完全完成，我们需要重新验证 g。如果我们在训练集中过拟合了模型，或者训练集太小或不具代表性，g 在测试集中可能有完全不同的解释。我们应该在测试集中评估准确性，以确保 g 在测试集中获得类似的准确度水平，且其解释与我们的意图一致。如果 g 在测试集中拟合得不好，很遗憾，我们不能回到训练数据或更改 g。然而，即使是一次失败的测试集验证，也向我们提供了关于未来实验中如何改进测量的重要信息。
划分样本带来一个显著的权衡——用于计算 ATE 的数据变少而导致的效率损失。这一点因我们不知道训练集需要多少数据而变得复杂——这就像是必须阐明"发现所需的功效（power）" (Egami et al., 2018)[^14]。我们希望该领域未来的工作能着手研究如何最有效地划分数据，以精炼概念并计算处理效应。
#### 24.5.3 Run Sequential Experiments

24.5.3  RUN SEQUENTIAL EXPERIMENTS
我们介绍的用文本数据进行因果推断的最后一种工作流是运行序贯实验。在这种设计中，研究者使用第一个实验的数据来发展 g，然后丢弃该实验的数据，并运行一个后续实验，在其中用 g 将文本映射到处理或结果变量。这种设计类似于运行一个试点研究，其中试点研究的数据被用来定义 g。在这种情况下，试点不仅用于设计调查或随机对照试验，还有助于固定分析。
序贯实验比样本划分提供了对研究发现可推广性更强的检验。跨序贯实验的结果复制，不仅要求 g 推广到另一个数据集，还要求因果效应推广到略有不同时期和人群。而随机训练/测试划分平均而言会反映相同的 ATE，跨序贯实验的 ATE 可能发生变化。因此，序贯实验是检验可推广性的理想方法，但需要更多资源和时间来实施。这一思想更一般的版本是研究跨情境和研究者的复制，其中序贯实验是作为科学家群体的一部分来执行的，而不是在单一特定研究中。

### 24.6 Conclusion

24.6  Conclusion
在本章中，我们首先回顾了因果推断的基本原理。我们讨论了预测、测量与因果推断之间的异同，并介绍了将因果推断用于文本的一些原则。然后我们描述了在与发现映射函数 g 相同的数据集中进行因果推断的问题，引入了潜在变量因果推断的根本问题（Fundamental Problem of Causal Inference with Latent Variables, FPCILV）(Egami et al., 2018)[^14]
